Adaptive Huffman File Reader & Writer
Status:
Complete
Framework:
Microsoft's .NET Framework and C#

Huffman encoding achieves file compression by using shorter binary-codes to represent the more frequent characters in a text file. To be honest, this little project showed me that you get much better compression with Zip files. Normal text file formats use fixed-length binary-codes to represent their characters; Huffman coding uses variable-length binary-codes to do this.

Huffman encoding encoding splits key generation and text encodign into seperate steps. This generates the need to find a way to get the key to the file recipient. Adaptive Huffman coding encodes the key into the end result as it's being compressed.

  • Created an Adaptive Huffman Tree class to be used for for text encoding and decoding
    • This is based on the Adaptive Huffman Tree I implemented for the Advanced Data Structures course at WSU-TC
  • Created a File Writer class that can be used for file compression
  • Created a File Reader class that can be used for file decompression
  • Source code can be found on GitHub

Periodic Table Selector
Status:
Complete
Framework:
WPF and C#

This is a Custom Selector Control built on the WPF Framework. It's in the shape of the Periodic Table of Elements. It's a recreation of one of the components I built for Sharc. I was able to make a performance improvment or two, simply because I had a bit more skill with WPF when I built this version. It doesn't have all the features of the Sharc version because all of those features were developed for Sharc's needs. I also added a feture or two that wasn't in Sharc, but would have been a bit helpful if they had occured to me back then. By default, selected chemical elements will turn on their glow effects to show that they've been selected. Yet, since is a selector control, there is also a bindable collection of selected elements that is kept up to date in real time.

I recreated this control for two reasons. The first being that my co-workers had a bit too much fun using this. Really, I walked into a co-worker's office more than one time, just to find him trying to make different shapes by selecting different elements. The other reason is that this might be the one thing I created while working at PNNL that I can re-create without getting into trouble.

  • Multiple forms of selection and deselection for chemical elements
    • Chemical elements can be selected by clicking on them through the GUI
    • Chemical elements can be selected by setting their IsChecked property in code
    • Chemical elements can be selected by directly inserting them into the PeriodicTable's SelectedElements property
    • Chemical elements can have their selection state changed from within any Chemical Group contain said elements
  • Chemical elements are designed to have their foreground, background, and glow-effect colors changed by assigning a different Brush object to those propeties
  • Chemical elements can be placed in Chemical Groups, allowing changes to be made to multiple elements at once
    • The Starndard Table Factory already has a few default Chemical Groups that are fairly standard in most Periodic Tables
    • Elements can belong to multiple Chemical Groups
    • Chemical Groups are able to do the following actions to all elements within the group:
      • Select chemical elements
      • Deselect chemical elements
      • Invert chemical element selection state
      • Set glow brush
      • Set background brush
      • Set foreground brush
  • The source code can be found on GitHub
    • The executables can also be found on GitHub
    • A demo project is included that shows off a few different ways this selector can be used
